# 安装与使用 nvm | nrm 和 yarn

目前主流的 node 版本管理工具有两种，nvmopen in new window 和 nopen in new window。 两者差异挺大的，具体分析可以参考一下淘宝 FED 团队的一篇文章：管理 node 版本，选择 nvm 还是 n？open in new window。

这里选择 nvm 管理 node 版本。

# 安装 nvm

安装可以参考nvmopen in new window项目的 README 文档，这里使用 curl 命令安装。

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
1

默认自动将对应的配置写入了~/.zshrc中，内容为：

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion
1
2
3

可以通过命令 source ~/.zshrc 加载，或者重新打开一个终端。

执行上面的命令时请在用户家目录执行。进入用户家目录的命令为：cd ~，检查是否为用户家目录的命令 pwd，显示为：~/你的用户名。

# 简单使用 nvm

# 查看可供使用稳定版本

nvm ls-remote
1

NodeJS 的版本发布很频繁，一般可以在NodeJS 官网open in new window看到 LTS 版本进行下载。

# 查看本地已经有的 NodeJS

nvm list
1

# 安装指定版本的 NodeJS

nvm install v16.13.0
1

# 指定默认的 NodeJS 版本

nvm alias default v16.13.0 # 指定默认的NodeJS版本
nvm use default
1
2

如果是切换不同版本的node，请确保对应的依赖已经安装，比如yarn，因为不同版本的node，对应的bin目录不一致，目录为：~/.nvm/versions/node/v16.13.0/bin。

# 查看系统当前使用的 NodeJS 版本

nvm current
1

nvm管理的node安装的目录在 ~/.nvm/versions/node/ 下。

# 卸载指定的 NodeJS 版本

nvm uninstall v16.13.0 // 卸载v16.13.0
1

至此，nvm 相关的基本操作完成，更多的操作请查看nvmopen in new window项目的 README 说明。

# nrmopen in new window

nrm 是 npm 镜像仓库管理命令，可以帮助我们管理 npm 镜像资源的下载地址。

# 安装

npm install -g nrm
1

鉴于国内网络环境，如果下载速度慢或者下载失败，建议修改 npm 的从淘宝镜像地址下载资源，可以执行命令：npm install -g nrm --registry=https://registry.npm.taobao.org/

# 简单使用 nrm

# 查看可用源

nrm ls
1

# 使用指定源

nrm use taobao
1

# 检查当前使用源

nrm current
1

此时，在 ~/.npmrc 配置中自动配置：registry=https://registry.npm.taobao.org/

# 安装 yarn

参考这里open in new window

npm -g install yarn
1

# 安装全局依赖

yarn global add cross-env live-server
1
